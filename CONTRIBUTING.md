# Contributing to Gizmo Chat Module

Thank you for your interest in contributing to Gizmo Chat Module! This document provides guidelines and instructions for contributing.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub with:

1. **Clear title and description**
2. **Steps to reproduce** the bug
3. **Expected behavior** vs **Actual behavior**
4. **Environment details** (PHP version, CodeIgniter version, browser, etc.)
5. **Screenshots** (if applicable)
6. **Error messages** from browser console or server logs

### Suggesting Features

We welcome feature suggestions! Please open an issue with:

1. **Clear description** of the feature
2. **Use case** - why this feature would be useful
3. **Possible implementation** (if you have ideas)

### Pull Requests

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
4. **Test thoroughly** - ensure your changes work and don't break existing functionality
5. **Update documentation** if needed (README, API docs, etc.)
6. **Commit with clear messages**: `git commit -m "Add feature: description"`
7. **Push to your fork**: `git push origin feature/your-feature-name`
8. **Open a Pull Request**

### Code Style

- Follow PSR-12 coding standards for PHP
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and single-purpose
- Maintain existing code style and formatting

### Testing

Before submitting a PR, please:

1. Test your changes in multiple browsers
2. Test on mobile devices
3. Verify no JavaScript console errors
4. Test with different user scenarios
5. Ensure database migrations work correctly
6. Test backward compatibility

### Documentation

If you add features or change behavior:

1. Update `README.md` if needed
2. Update `CHANGELOG.md` with your changes
3. Update `docs/API.md` if API endpoints changed
4. Update `docs/CONFIGURATION.md` if configuration changed
5. Add code comments for new functions

### Questions?

If you have questions about contributing, please open an issue with the `question` label.
